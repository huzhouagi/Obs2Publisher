# Obs2Publisher

Obs2Publisher 是一个桌面版 Obsidian 社区插件，用于在 Obsidian 内将当前 Markdown 笔记转换为微信公众号、X Articles 和小红书可用的内容。

## 功能

- 微信公众号：8 套内联样式主题，富文本复制，普通图片 Base64 内嵌，HTML 导出。
- X Articles：标题与正文合并复制，普通图片按原位置生成编号占位符，表格自动转为高清截图，可一键导出全部原图与表格截图，并直接打开 X Articles 后台。
- 小红书：5 套 3:4 H5 卡片主题，自动提炼 4–6 张（最多 8 张），保留正文图片、表格和列表，概要支持 Markdown 编辑，并可单页下载或批量下载全部 1080×1440 PNG 截图。
- 当前笔记实时预览，不调用 AI，不上传笔记，不收集遥测。

## 安装

将发布包中的 `main.js`、`manifest.json`、`styles.css` 放入：

```text
<Vault>/.obsidian/plugins/obs2publisher/
```

在 Obsidian 的“设置 → 第三方插件”中启用 Obs2Publisher。点击左侧纸飞机图标，或从命令面板运行“打开内容分发面板”。

## 使用

1. 打开一个 Markdown 笔记。
2. 打开右侧 Obs2Publisher 面板。
3. 选择目标平台并检查实时预览。
4. 使用底部按钮复制正文、导出 HTML、下载图片或生成 ZIP。

### 公众号主题

内置极简黑白、蓝灰科技、暖红编辑部、石墨留白、薄荷清单、橄榄手记、雾蓝刊读和杏仁暖刊 8 套主题。新增主题遵循“灰阶承重、主色点睛、细线与留白分层”的轻量设计，不依赖外部 CSS。所有主题统一压缩顶部留白，杏仁暖刊的一级标题使用左对齐。

公众号顶部支持系统默认、中文黑体、中文宋体、现代圆体和刊物衬线 5 种字体。底部提供“复制内容”和“打开后台”操作。微信公众号、X Articles 和小红书输出都会自动忽略笔记开头的 YAML 属性信息。

微信公众号和 X Articles 输出始终包含文章标题；原文没有一级标题时使用笔记文件名。X Articles 底部提供“复制”“导出图片”和“打开 X 文章”，图片按文章顺序写入导出目录，包含正文原图和表格生成的 PNG 截图。

### 小红书分页

- 小红书统一使用章节分页，优先按 Markdown 章节提炼要点。
- 内容采用本地抽取式压缩，不调用 AI，不生成原文没有的事实。
- 一般生成 4–6 张卡片，长文最多 8 张；从第一页开始统一按正文顺序分页，不再额外生成整篇摘要封面，也不重复第二页内容。
- 正文默认字号为 14px，可在设置中按 10–20px 调整；只在完整句子和完整内容块边界精简，不按字符截断。
- 章节分页优先提取关键章节，并直接使用原文章节名作为卡片标题。
- 正文图片和表格属于不可丢失内容；长表格每 6 行分成一个带重复表头的完整 Markdown 表格块。
- 顶部主题右侧提供与公众号一致的 5 种字体选择。卡片内容概要默认折叠，点击“展开”后可修改每页标题和完整 Markdown 正文，支持图片、表格、列表、代码块和行内代码，修改实时同步至预览和 PNG/ZIP 导出。拖动任一正文编辑框会同步所有概要卡片的编辑高度。
- 所有卡片标题统一为“正文字号 + 2px”，正文使用设置中的统一字号；图片和字体加载完成后会按真实卡片高度静默适配，保留底部安全空间，必要时同步精简概要和卡片中的完整正文块；图片仅展示画面，不在下方重复显示图片标题。
- 正文图片始终居中并保持原始比例，会根据当页文字量和图片数自适应宽高；当内容较少时优先放大展示。
- 默认内置“胡诌AGI”头像，用户名为“胡诌AGI”，用户 ID 为“@huzhouagi”；三者可在设置中修改或清空，并按“头像、用户名、用户 ID”的顺序显示在每页左下角，与右侧页码同行。已设置头像时，设置页会在“更换头像”左侧显示可点击预览的缩略图。
- 普通段落默认保持自然段；只有三个及以上并列要点时才自动使用无序列表，有序列表沿用原文结构。Emoji 仅用于图表或图片等明确内容场景。
- “编辑系统”模板统一使用米白底色、黄色/蓝色点睛和大号页码语言，同一主题下不再交替明暗样式。

### 图片规则

- 支持 Vault 本地图片、相对路径图片、Wiki 图片、Base64 图片和 HTTP/HTTPS 外链图片。
- 公众号普通图片会转为 Base64 后与 HTML 一同复制；GIF 需要在公众号后台手动上传。
- X Articles 普通图片使用编号占位符；表格保留在复制区域中并转换为 Base64 PNG 截图。“导出图片”会同时输出原图和表格截图。
- 外链图片只有在当前笔记实际引用时才会请求；失效图片会在面板或导出清单中报告。

插件设置中可分别自定义公众号文章、X Articles 和小红书的 Vault 内相对导出路径。留空时默认写入 `Obs2Publisher Exports/公众号文章`、`Obs2Publisher Exports/X Articles` 和 `Obs2Publisher Exports/小红书`。成功提示会显示实际文件路径，小红书 PNG 仍会校验为 1080×1440。

三个平台的底部操作区左侧都提供“关于作者”和“设置”入口。“关于作者”中可查看作者简介、公众号、X、GitHub、小红书主页及赞赏码；“设置”可直接打开当前插件设置。

## 隐私

插件不要求账号、不包含遥测、不把笔记内容发送到作者或第三方服务器。唯一的网络请求是读取当前笔记中明确引用的外链图片。

## 已知限制

- 仅支持桌面版 Obsidian 1.7.2 及以上版本。
- 不直接登录或发布到微信公众号、X 或小红书。
- 公众号、X 等第三方编辑器可能随时调整粘贴规则，发布前应检查最终效果。
- 极端密集的表格、代码块和大量长图仍建议在导出前逐页检查。

## 开发

```bash
npm run dev
npm run check
npm run build
```
